# Contributing to Neuradash 🚀

First off, thank you for considering contributing to Neuradash! It's people like you that make Neuradash such a powerful tool for the community.

Please note that while this project is public, it is a **Proprietary Project**. Contributions are welcome but will be subject to our internal review and licensing terms.

## 🌈 Code of Conduct

We aim to maintain a professional and welcoming environment. Please be respectful and constructive in all interactions.

## 🛠️ How Can I Contribute?

### Reporting Bugs
*   Check the [Issues](https://github.com/yogisyahroni/TOOLS_BI/issues) to see if the bug has already been reported.
*   If not, open a new issue with a clear title, description, and steps to reproduce.

### Suggesting Features
*   Open an issue with the tag `enhancement`.
*   Explain the "Why" behind the feature and how it would benefit the users.

### Pull Requests
1.  **Fork** the repository.
2.  Create a **Feature Branch** (`git checkout -b feat/amazing-feature`).
3.  **Commit** your changes following [Conventional Commits](https://www.conventionalcommits.org/).
4.  **Push** to the branch (`git push origin feat/amazing-feature`).
5.  Open a **Pull Request**.

## 🎨 Coding Standards

*   **TypeScript**: Use strict types. Avoid `any`.
*   **React**: Prefer Functional Components and Hooks.
*   **Styling**: Use Tailwind CSS classes. Keep it clean and responsive.
*   **Git**: Use descriptive commit messages (e.g., `feat(ui): add new dashboard widget`).

## 🏗️ Setup for Development

1.  Clone the repo: `git clone https://github.com/yogisyahroni/TOOLS_BI.git`
2.  Install dependencies: `npm install`
3.  Set up environment: `cp .env.example .env` (and fill in the values)
4.  Run development server: `npm run dev`
